# Ustatic CSS Playground - Vue

Интерактивная демонстрация различных типов инициализации плагина `ustaticCss` во Vue 3.

## 🚀 Запуск

```bash
# Установка зависимостей
npm install

# Запуск dev-сервера
npm run dev

# Сборка для продакшена
npm run build
```

## 📖 Возможности

Playground позволяет экспериментировать с тремя типами инициализации:

### 1. **Режим модулей** (Modules)
Явное указание модулей для подключения:

```js
app.use(ustaticCss, {
  modules: ['flexbox', 'spacing', 'typography'],
  loadVars: true,
  loadBase: true
})
```

**Когда использовать:**
- Хотите явно контролировать какие модули загружать
- Знаете заранее, какие стили понадобятся
- Оптимизируете размер бандла

### 2. **Режим автоподбора** (Classes)
Автоматическое определение модулей по используемым классам:

```js
app.use(ustaticCss, {
  classes: ['flex', 'p-4', 'text-lg', 'bg-primary']
})
```

**Когда использовать:**
- Быстрый прототипинг
- Не хотите вручную перечислять модули
- Хотите чтобы библиотека сама определила зависимости

### 3. **Ручной режим** (Manual)
Полный контроль через программный API:

```js
app.use(ustaticCss, { /* ваши опции */ })
// или
await loadStyles({ modules: ['flexbox'] })
```

**Когда использовать:**
- Нужна кастомная логика загрузки
- Динамическая подгрузка модулей
- Интеграция с другими библиотеками

## 🧩 Компоненты

### ConfigurationPanel
Панель конфигурации с настройками:
- Выбор типа инициализации (radio-кнопки)
- Список модулей (checkbox) для режима модулей
- Поле ввода классов для режима автоподбора
- Текстовое поле для ручного кода
- Дополнительные опции (CSS-переменные, базовые стили)

### InitCodePreview
Показывает сгенерированный код инициализации с:
- Подсветкой синтаксиса
- Кнопкой копирования
- Контекстными подсказками

### DynamicApp
Динамически создаваемое Vue-приложение:
- Пересоздаётся при изменении конфигурации
- Показывает статус инициализации
- Выводит логи операций

### DemoApp
Демонстрационное приложение с примерами:
- Flexbox раскладки
- Типографика
- Отступы и размеры
- Интерактивные элементы
- Позиционирование
- Границы и эффекты

## 🎯 Как это работает

1. **Конфигурация** хранится в реактивном состоянии `App.vue`
2. При изменении конфигурации `DynamicApp` получает новые props
3. `DynamicApp` уничтожает предыдущее приложение и создаёт новое
4. Новый инстанс `createApp()` инициализируется с указанной конфигурацией
5. `InitCodePreview` показывает актуальный код для копирования

## 📁 Структура

```
src/
├── components/
│   ├── ConfigurationPanel.vue    # Панель настроек
│   ├── InitCodePreview.vue       # Предпросмотр кода
│   ├── DynamicApp.vue            # Динамическое приложение
│   ├── DemoApp.vue               # Демо-контент
│   ├── CodeEditor.vue            # Редактор HTML
│   └── Preview.vue               # Предпросмотр HTML
├── App.vue                       # Главный компонент
└── main.ts                       # Точка входа
```

## 💡 Советы

- **Экспериментируйте** с разными конфигурациями чтобы понять разницу
- **Смотрите логи** в `DynamicApp` для понимания процесса инициализации
- **Копируйте код** из `InitCodePreview` для использования в своих проектах
- **Тестируйте** в HTML-редакторе разные классы для проверки работы стилей

## 🔧 Расширение

Для добавления новых примеров в `DemoApp.vue`:

```vue
<template>
  <div class="demo-app">
    <!-- Ваш пример -->
    <div class="p-4 bg-primary text-white rounded">
      Ваш контент
    </div>
  </div>
</template>
```
